# 安全政策

## 支持的版本

| 版本 | 支持状态 |
| --- | --- |
| 0.5.x | ✅ 支持 |
| < 0.5 | ❌ 不支持 |

## 报告漏洞

如果你发现了安全漏洞，请**不要**在公开的 Issue 中报告。

### 报告方式

1. **GitHub Security Advisory**（推荐）：
   - 前往 [Security Advisories](../../security/advisories)
   - 点击 "New draft security advisory"
   - 填写漏洞详情

2. **私信联系**：
   - 通过 GitHub Discussions 私信项目维护者

### 报告内容

请在报告中包含以下信息：

- 漏洞类型（如：信息泄露、注入、权限提升等）
- 受影响的组件/文件路径
- 复现步骤
- 潜在影响评估
- 如有可能，提供修复建议

### 响应时间

- 确认收到：48 小时内
- 初步评估：7 天内
- 修复发布：视严重程度，通常 14-30 天

## 安全最佳实践

使用 VideoForge 时，请注意以下安全事项：

### API Key 保护

```bash
# ✅ 正确：使用 .env 文件（已在 .gitignore 中）
cp .env.example .env
# 编辑 .env 填入 API Key

# ❌ 错误：硬编码在代码中
# ❌ 错误：提交到版本控制
```

### 素材来源

- 通过 yt-dlp 下载的素材请遵守原作者版权
- 商用场景请确保素材授权
- 建议使用 CC0/CC-BY 等开放授权素材

### 本地数据

- SQLite 数据库存储在 `data/` 目录
- FAISS 索引包含素材的向量表示
- 建议对敏感项目使用独立的数据目录

## 已知安全考虑

| 组件 | 风险 | 缓解措施 |
|------|------|----------|
| LLM API 调用 | API Key 泄露 | 使用 .env，不提交敏感配置 |
| yt-dlp 下载 | 下载恶意内容 | 仅从可信源下载 |
| HyperFrames 渲染 | 模板注入 | 审查自定义模板 |
| 本地文件访问 | 路径遍历 | 使用 `utils/paths.py` 进行路径验证 |
