# VideoForge 开发路线图 (Roadmap)

## 项目愿景

面向知识区 UP 主的"本地优先"视频自动化工作台。核心差异化：**不用 AI 生成画面，而是智能检索和组合真实素材**，保证学科严谨性。

---

## 实施原则

> **先验证核心表现力，再补全智能链路，最后实现产品化**

---

## Phase 1: 渲染闭环验证 ✅ 已完成

**目标**：验证 HyperFrames 能否渲染出"花生视频"风格的成片

**交付物**：
- [x] HTML/CSS 视频模板（动态字幕、梗图飞入、缩放转场）
- [x] HyperFrames 渲染器集成
- [x] 手动数据 → MP4 输出验证

**关键成果**：确认 HyperFrames + GSAP 动效方案可行

---

## Phase 2: 竞品 RAG 编剧链路 ✅ 已完成

**目标**：跑通"竞品字幕 → 提取结构 → 生成新脚本"的 RAG 链路

**交付物**：
- [x] `ScriptWriterSkill` - DeepSeek RAG 编剧
- [x] `TTSSkill` - Edge-TTS 配音生成
- [x] `TimestampAlignSkill` - WhisperX 字级时间戳
- [x] 结构化 JSON 分镜表输出

**关键成果**：LLM 编剧 + TTS + 时间戳对齐流程打通

---

## Phase 3: 云端素材检索 ✅ 已完成

**目标**：实现云端素材自动获取（绕过本地冷启动问题）

**交付物**：
- [x] `YtdlpProvider` - YouTube/Bilibili 素材下载
- [x] 英文关键词 Prompt 优化
- [x] 降级策略（纯色占位图）

**关键成果**：云端素材检索链路可用

---

## Phase 4: 端到端组装与渲染 ✅ 已完成

**目标**：一键输出示例科普视频

**交付物**：
- [x] `PipelineRunner` 核心调度器
- [x] 脚本 → 配音 → 素材 → 渲染 全链路串联
- [x] `test_e2e.py` 端到端测试

**关键成果**：知识点 → MP4 全自动流水线

---

## Phase 5: 资源库基础设施 ✅ 已完成

**目标**：建立本地素材管理系统

**交付物**：
- [x] SQLite 数据库 (`storage/database.py`)
- [x] JSON Sidecar 元数据 (`storage/sidecar.py`)
- [x] 路径工具 (`utils/paths.py`)

**关键成果**：本地素材可持久化管理

---

## Phase 6-7: CLIP 嵌入与 FAISS 向量检索 ✅ 已完成

**目标**：让本地素材"可被 AI 检索"

**交付物**：
- [x] `CLIPEmbedder` - 视频帧向量化
- [x] `VectorStore` - FAISS 索引管理
- [x] `LocalFaissProvider` - 本地优先检索

**关键成果**：本地 FAISS 向量检索可用

---

## Phase 8-9: 自动入库与 RAG 模板 ✅ 已完成

**目标**：自动化素材入库 + 叙事结构模板库

**交付物**：
- [x] `DedupService` - 重复检测
- [x] 自动切片入库流程
- [x] `RAGTemplateExtractor` - 叙事结构提取
- [x] 模板库存储

**关键成果**：素材自动入库 + RAG 模板积累
